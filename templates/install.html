Installation

Instructions on getting up and running with Kanso's command-line

  1. CouchDB

    Since Kanso apps run on CouchDB, you'll need to have CouchDB installed. It is strongly recommended you have at least CouchDB v1.1. There were significant improvements introduced in that release for Kanso apps, and many packages will not even work on earlier versions.

    Get CouchDB at couchdb.org

    Note: Couchbase Single Server 1.1 is not based on Apache CouchDB v1.1.

  2. Node.js

    Officially, Kanso supports the latest stable version of node.js. However, the v0.6.x releases have a bug preventing Kanso exiting after some commands. Until this issue is fixed it's recommended you use the v0.4.x series.

    Get Node.js at nodejs.org

    Why does Kanso need node.js?

    Node.js is used for the command-line tools only. With Kanso, the end result is a pure CouchApp you can host using CouchDB alone. Using node to write the associated tools allows us to do some powerful things by interpreting the JavaScript of your application.

  3. Kanso command-line

    First, clone the Kanso repository from GitHub: 

    git clone git://github.com/kanso/kanso.git
cd kanso

    Then, you can fetch it's dependencies and install by doing: 

    make && sudo make install
